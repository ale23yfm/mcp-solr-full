1. Project Overview

MCP Solr is a Model Context Protocol (MCP) server that provides a bridge between AI assistants and Apache Solr. Built in pure PHP with no external dependencies, it enables CRUD operations on Job and Company data cores.


Project Status: ACTIVE - All 11 tests passing

Key Features

  • Full CRUD Operations - Create, Read, Update, Delete for Job and Company cores
  • Flexible Search - Support for complex queries with filters
  • JSON-RPC 2.0 - Industry-standard protocol over STDIO
  • OpenCode Compatible - Ready-to-use MCP configuration
  • Zero Dependencies - Pure PHP with cURL only
  • Docker Ready - Containerized deployment

Technology Stack

Search Engine Apache Solr 9.x

Protocol Flow

  1. Client sends JSON-RPC request via STDIN
  2. MCP Server parses request and validates method
  3. Server calls appropriate function from functions.php
  4. Function makes HTTP request to Solr API
  5. Solr responds with JSON
  6. Server formats response and sends via STDOUT

4. Configuration

MCP Solr uses environment variables for configuration. These can be set when running the Docker container or in your local environment.

Variable Default Required Description
SOLR_HOST localhost No Solr hostname or IP address
SOLR_PORT 8983 No Solr port number
SOLR_USER solr No Solr username for Basic Auth
SOLR_PASS SolrRocks No Solr password for Basic Auth
SOLR_SCHEME http No HTTP or HTTPS

Docker Configuration

# Build the Docker image
docker build -t mcp-solr .

# Run with default configuration (connects to localhost:8983)
docker run --rm -i --network host mcp-solr

# Run with custom environment variables
docker run --rm -i --network host \
  -e SOLR_HOST=127.0.0.1 \
  -e SOLR_PORT=8983 \
  -e SOLR_USER=solr \
  -e SOLR_PASS=SolrRocks \
  -e SOLR_SCHEME=http \
  mcp-solr

OpenCode Integration

Add this to your opencode.json:

{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "mcp-solr": {
      "type": "local",

                  "-e", "SOLR_PORT=8983", 
                  "-e", "SOLR_USER=solr", 
                  "-e", "SOLR_PASS=SolrRocks", 
                  "mcp-solr"],

    }
  }
}

7. API Reference

MCP Protocol Methods

initialize

Establish connection and negotiate capabilities with the MCP client.

// Request
{"jsonrpc": "2.0", "method": "initialize", "params": {"protocolVersion": "2024-11-05"}, "id": 1}

// Response
{"jsonrpc": "2.0", "id": 1, "result": {
  "protocolVersion": "2024-11-05",
  "capabilities": {"tools": {}},
  "serverInfo": {"name": "mcp-solr", "version": "1.0.0"}
}}

tools/list

List all available tools/methods.

// Request
{"jsonrpc": "2.0", "method": "tools/list", "params": {}, "id": 2}

ping

Health check - returns empty result on success.

// Request
{"jsonrpc": "2.0", "method": "ping", "params": {}, "id": 3}

Job Core Methods

job_select - Search Jobs

job_select(query = '*:*', start = 0, rows = 10, filters = [])
query string Solr query string (default: *:*)
start int Pagination offset (default: 0)
rows int Number of results (default: 10)
filters array Optional: fields, status, company, workmode, location 
{"jsonrpc": "2.0", "method": "job_select", "params": {"query": "PHP", "rows": 5}, "id": 1}

job_search - Simplified Search

job_search(term = '*:*', offset = 0, limit = 20, filters = {}) 
{"jsonrpc": "2.0", "method": "job_search", "params": {"term": "developer", "status": "published"}, "id": 2}

job_insert / job_index - Insert Job

job_insert(data) | job_index(data)
url string REQUIRED - Unique job URL
title string Job title
company string Company name
location array Array of locations
tags array Array of skill tags
workmode string remote, on-site, or hybrid
salary string Salary range (e.g., "5000-8000 RON")

10. Data Models

Job Model (unique key: url)

Field Type Required Description
url string YES Full URL to job detail page (unique key)
title string Recommended Position title (max 200 chars, no HTML, trimmed)
company string No Hiring company name (legal name, diacritics required)
cif string No Company CIF/CUI
location array No Romanian cities/addresses (multi-valued)
tags array No Skills/tags (lowercase, max 20, no diacritics)
workmode string No remote, on-site, or hybrid
date date No UTC ISO8601 scrape timestamp
status string No scraped, tested, published, verified
vdate date No Verified date (ISO8601)
expirationdate date No Job expiration (vdate + 30 days max)
salary string No Format: "MIN-MAX CURRENCY" (e.g., "5000-8000 RON")

Company Model (unique key: id)

Field Type Required Description
id string YES CIF/CUI (8 digits, no RO prefix - unique key)
company string YES Legal name from Trade Register (diacritics required)
status string No activ, suspendat, inactiv, radiat
location array No Romanian cities/addresses (multi-valued)
website array No Official website URLs (valid HTTP/HTTPS)
career array No Career page URLs (valid HTTP/HTTPS)

Solr Core Configuration

Note: The Job and Company cores must be pre-configured in Apache Solr with the appropriate schema fields. This MCP server assumes the cores are already created.
